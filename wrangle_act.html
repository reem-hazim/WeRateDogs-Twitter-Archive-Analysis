Quality Issues¶

twitter_archive¶

  • Incomplete data (NaNs in the columns "in_reply_to_status_id", "in_reply_to_user_id", "retweeted_status_id", "retweeted_status_user_id", and "retweeted_status_timestamp".)
  • Missing expanded URLs
  • Missing and incorrect dog names such as "this", "a" and "an".
  • Missing and incorrect dog stages
  • Missing image predictions (Note: Since we don't have the algorithm for the image predictions, we can't do anything about this issue)
  • Erroneous data type: timestamp is a string instead of dateTime
  • Some tweets have invalid rating denominators and numerators.
  • Some of the tweets are actually retweets. (181 tweets have non-null values for "retweeted_status_id")

image_predictions¶

  • Some of the tweets are not about dogs, as indicated by the all the predictions being not dog breeds (False).
  • Columns don't have descriptive names

Tidiness Issues¶

  • The stages of dogs (doggo, floofer, pupper, puppo) are in wide format. To meet the tidy data requirements, they have to be in long format.
  • additional_tweet_info and twitter_archive should be one table instead of two.
  • We can merge the image_predictions table with twitter_archive as well
  • Some tweets have multiple expanded_urls separated by commas

After fixing tidiness issues, new quality issues arise:¶

  • The favorite_count and retweent_count columns became floats after the merge with twitter_archive_clean
  • There are 3 NA values for retweet_count and favorite_count. Probably some of the tweets that were deleted.
  • dog_stage should have the data type of category
  • Many duplicate values resulting from pd.melt()
